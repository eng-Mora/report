Instruction Cycle and Von Neumann Architecture

In this report, we will thoroughly explain the concept of the Instruction Cycle, which is considered the core of any operation within a computer. We will also discuss the key elements of the classical architecture (Von Neumann Architecture) and understand how software programs are translated into instructions that control the hardware using the Control Unit (CU).

1. Overview of Computer Functions and Component Connections

A computer consists of interconnected components that work together to execute programs. These components include:

  • Control Unit (CU)
  • Arithmetic Logic Unit (ALU)
  • Main Memory
  • Input/Output Units (I/O Units)

2. Von Neumann Architecture

Most modern computers are based on this architecture, which is built on three key principles:

  1. Storing data and instructions in the same readable/writable memory.
  2. Accessing any memory location using a direct address, regardless of content type.
  3. Executing instructions sequentially, unless altered by control instructions (e.g., jump).

3. Program Concept

There are two approaches to building systems that perform specific tasks:

A. Special-Purpose Hardware

  • Logical components are connected in a specific way to perform one dedicated task.
  • To change the task, hardware rewiring is required → not flexible.

B. General-Purpose Hardware

  • A general system is built and programmed using control signals sent through software commands.
  • Flexible, as functionality can be changed by altering the program.

4. What is a Program?

A program is simply:

  • A sequence of steps,
  • Each step performs an arithmetic or logical operation,
  • Each operation is linked to a specific opcode (e.g., ADD = 9D, MOVE = E3).

The Control Unit (CU) receives the opcode and generates the corresponding control signals.

5. Main Computer Components and Their Functions

  • Control Unit (CU): Interprets instructions and generates control signals.
  • Arithmetic Logic Unit (ALU): Performs logical and arithmetic operations.
  • Main Memory (MM): Temporarily stores data and instructions.
  • Input/Output Units (I/O): Interact with the user and peripheral devices.

6. The Instruction Cycle

It consists of two main phases:

A. Fetch Cycle

Program Counter (PC): Holds the address of the next instruction.

Steps:

  1. The processor reads the instruction from the memory location pointed to by the PC.
  2. The instruction is loaded into the Instruction Register (IR).
  3. The PC is incremented automatically.

B. Execute Cycle

The processor interprets the instruction in the IR and performs one of the following based on the instruction type:

  • Transfer data between memory and CPU.
  • Execute arithmetic or logical operations.
  • Transfer data to/from I/O units.
  • Control the program flow (e.g., jump).

7. Graphical Representation of the Instruction Cycle

An example is used to illustrate the cycle:

Instruction: Add locA, R0
Assumptions: Memory at locA = 7, R0 = 10

Execution Steps:

  1. PC = 1200 → Instruction is fetched.
  2. Instruction is loaded into IR.
  3. Interpreted as: "Add the value at locA to R0."
  4. Value at locA is fetched from memory.
  5. Operation executed: R0 = R0 + locA = 10 + 7 = 17

8. Hypothetical Machine

An example of a hypothetical machine is introduced to demonstrate how instructions are built:

  • Each instruction consists of an opcode and a memory address.

Examples of Instructions:
0001: Load AC from memory
0101: Add to AC from memory

Important CPU Registers:

  • PC (Program Counter)
  • IR (Instruction Register)
  • AC (Accumulator)

The instruction cycle is the heart of how a computer operates. By understanding how an instruction is fetched, decoded, and executed, we gain insight into how programs work behind the scenes. This foundational knowledge is essential in the study of computer architecture, performance enhancement, and processor development.
